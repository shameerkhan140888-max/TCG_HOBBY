# UI Principles

## 1. Retail Clarity
Every screen should help a customer find, understand, or complete a task quickly. The interface should support shopping, collecting, and managing stock without distraction.

## 2. Premium Restraint
The product should feel polished without becoming decorative. Use strong spacing discipline, subtle depth, and controlled colour usage.

## 3. Operational Confidence
Administrative and commerce screens should feel reliable and audit-friendly. Staff should be able to scan, compare, and act without friction.

## 4. Information Hierarchy
- Primary actions should be obvious.
- Supporting details should stay present but quiet.
- Tables, cards, and forms should clearly separate labels, values, and actions.
- Important numbers should be easy to scan.

## 5. Consistency Over Novelty
- Reuse patterns rather than inventing new ones.
- Keep buttons, badges, cards, and form fields aligned to the same visual language.
- Use the same visual treatment for similar states across the product.

## 6. Accessibility by Default
- Design for keyboard and screen reader use from the start.
- Never hide essential actions behind unclear icons alone.
- Use readable contrast, focus states, and predictable structure.

## 7. Responsive Grace
- Mobile experiences should remain fully usable, not simplified into dead ends.
- Desktop experiences should feel richer, not noisier.
- Layouts should adapt without changing the mental model.

## 8. Honest Motion
- Motion should help users understand change.
- Menus, drawers, overlays, and hover states may animate softly.
- Do not use animation to distract from poor structure.

## 9. Content Discipline
- Keep labels concise.
- Avoid hype language.
- Use plain commerce terms.
- Avoid making claims about value or outcomes that the product does not provide.

## 10. Acceptance Mindset
Before a screen ships, it should answer:
- Is it easy to understand?
- Is it fast to scan?
- Is it clearly clickable?
- Does it work on mobile and desktop?
- Does it look like a real product?

If the answer to any of those is no, the screen is not ready.
